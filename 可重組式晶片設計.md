##  SPI通訊協定
### 基本介紹:

SPI 是一種高速、全雙工的同步串列通訊協定，採主從架構，主要由四條線控制。
![image](https://github.com/Ericfishcan666/Reconfigurable-Chip/blob/main/1.png)

傳輸模式:
![image](https://github.com/Ericfishcan666/Reconfigurable-Chip/blob/main/2.png)
因此CPOL、CPHA排列組合下來總共四種

傳輸格式:
![image](https://github.com/Ericfishcan666/Reconfigurable-Chip/blob/main/3.png)
從MSB開始接收或輸出的串列訊號
當command bit為1時代表進行讀取操作，反之為寫入操作

架構圖:
![image](https://github.com/Ericfishcan666/Reconfigurable-Chip/blob/main/4.png)

每個小方塊的介紹:
PLL:將原本的系統時脈提升至100M，在quartus內可以用他們的IP進行實作即可。
> 為何需要PLL? 為甚麼PLL可以把clk調更高?

LPF: 低通濾波器，只要是用來過濾高頻雜訊，留下乾淨低頻的原始訊號

SPI_RX: SPI傳輸協定的接收端

Register_file: 可用來讀或寫的暫存器

SPI_TX: SPI傳輸協定的發送端

### 開始實作
## LPF

主要負責的功能: 

過濾雜訊，把高頻部分去除。

輸入:

門檻值(以0.08us為一單位，100MHz的倒數)、原始的訊號

輸出:

乾淨的訊號

程式碼解說:

以q位移暫存器實作，必須連續r_LPF_threshold個clk來都維持一樣的訊號，若偵測到不一樣的訊號，馬上reset counter，否則繼續數下去直到(2^N-1)-1個CLK來。


```
module low_pass_filter
#(
    parameter int N = 14
)
(
    output logic sig_filter,
	 
    input  logic signal,
    input  logic [N-1:0] r_LPF_threshold, // Unit : 0.08us    
    input  logic clk,
    input  logic reset
);

		
	logic [N-1:0] counter;
	logic         reset_counter;
	logic         LPF_threshold;
	logic [4:0]   q;
	
	always@(posedge clk)begin
		if(!rst)begin
			LPF_threshold<=0;
			counter<=0;
			sig_filter<=signal;
			q<=0;
		end
		else begin
			q[4:0]<={q[3:0],signal};
			if(LPF_threshold) sig_filter<=q[4:0];
			reset_counter<= q[1]^q[0];
			LPF_threshold<=(counte>=r_LPF_threshold);
		end
	end
	
	always@(posedge clk)begin
		if(~rst|reset_counter) counter<=0;
		else if(~counter[N-1]) counter<=counter+1;
	end
		
			

endmodule

```
## SPI_RX
架構圖:
![image](https://github.com/Ericfishcan666/Reconfigurable-Chip/blob/main/5.png)

posedge 和 negedge detector

主要負責的功能: 
抓取posedge 和 negedge


輸入:
ssn、sclk


輸出:
ssn訊號的正緣、sclk訊號的負緣


程式碼解說:
如果negedge代表 previous state==1 & current state ==0

如果posedge代表 previous state==0 & current state ==1



```
module spi_rx_edge_detector (
    input  logic CLK_100MHz,
    input  logic reset,
    input  logic ssn,
    input  logic sclk,
	 
    output logic ssn_negedge,
    output logic sclk_posedge
);
    logic ssn_pre, ssn_curr; 
    logic sclk_pre, sclk_curr;
	 //define previous state & current state
	 

	 always@(posedge CLK_100MHz)begin
	  if(!reset)begin
			ssn_pre<=0;ssn_curr<=0;
			
			sclk_pre<=0, sclk_curr<=0;
		end
		else
			ssn_pre<=ssn_curr;
			ssn_curr<= ssn;
			
			sclk_pre<=sclk_curr;
			sclk_curr<=sclk;
		end
	end
	
	 //needge: previous state==1 & current state==0
	 //posedge: previous state==0 & current state=1
	assign ssn_negedge=(ssn_pre&~ssn_curr);
	assign sclk_posedge=(~sclk_pre&sclk_curr);
	
endmodule
```
counter

主要負責的功能: 紀錄目前應該收到幾個資料了



輸入:
sclk_posedge:sclk_posedge代表資料才會開始傳輸進來


輸出:
receive_data_counter紀錄目前應該收到幾個資料，這個參數在後面會常常被使用到


程式碼解說:

```
module spi_rx_counter (
    input  logic CLK_100MHz,
    input  logic reset,
    input  logic sclk_posedge,
    input  logic receive_data_counter_reset,
    output logic [31:0] receive_data_counter
);
    
	 always_ff(posedge CLK_100MHz)begin
		if(reset|receive_data_counter_reset)begin
			receive_data_counter<=0;
		end
		else if(sclk_posedge)begin
			receive_data_counter<=receive_data_counter+1;
		end
	 end
	 
endmodule
```
shift register

主要負責的功能: 在sclk_posedge來時，不斷的shift mosi訊號，直到遇到reset，清空shift register。

輸入: 
mosi: master傳給slave的串列訊號



輸出:
shift_data左移過後的訊號



```
module spi_rx_shift_register (
    input  logic CLK_100MHz,
    input  logic reset,
    input  logic mosi,
    input  logic sclk_posedge,
	 
    output logic [15:0] shift_data
);
	always_ff(posedge CLK_100MHz)begin
		if(reset)shift_data<=0;
		
		else if(sclk_posedge)shift_data[15:0]<={shift_data[14:0],mosi};
	end
    
endmodule
```
register_rx

主要負責的功能: 
單純讀取左移過後的資料，選擇要讀取哪個部分(address、command還是data)


輸入:
shift_data
和
load_address、load_command、load_data
是否將讀進來的資料存進相對應的暫存器

輸出:
相對應的暫存器:address、command、data這三個暫存器


程式碼解說:

```
module spi_rx_reg (
    input  logic CLK_100MHz,
    input  logic reset,
    input  logic [15:0] shift_data,
	 
    input  logic load_address,
    input  logic load_command,
    input  logic load_data,
	 
    output logic [7:0]  address,
    output logic        command,
    output logic [15:0] data
);
    
	 always_ff(posedge CLK_100MHz)begin
		if(reset)begin
			address<=0;
			command<=0;
			data<=0;
		end
		else begin;
			if(load_address)address<=shift_data[15:8];
			if(load_command)command<=shift_data[7];
			
			if(load_data)data<=shift_data[15:0];
		end
	 end
	 
endmodule
```
rx_fsm

狀態轉換圖:
![image](https://github.com/Ericfishcan666/Reconfigurable-Chip/blob/main/6.png)

主要負責的功能: 
當shift data經過左移到達一定的位置時，再根據現在的狀態讀取資料，來做進一步的動作，例如要讀或寫，假如要讀，則會把adress送給register file取出資料、並且將TX_req設為1，啟用接下來發送的動作。假如是要寫入，則會等後面的資料傳進來，再寫入指定位置。



程式碼解說:

```
module spi_rx_fsm (
    input  logic CLK_100MHz,
    input  logic reset,
    input  logic ssn_negedge,
    input  logic sclk_posedge,
    input  logic [31:0] receive_data_counter,
    input  logic command,  // 從 reg 接過來
    
    output logic receive_data_counter_reset,
    output logic load_address,
    output logic load_command,
    output logic load_data,
    output logic write_en,
    output logic read_en,
    output logic tx_req,
    output logic rx_finish
);
    // 狀態定義
    typedef enum logic [1:0] {IDLE, WAIT_HEADER, WAIT_DATA} state_t;
    state_t state;

    
	 always_ff(posedge CLK_100MHz)begin
		if(reset)begin
			  receive_data_counter_reset<=0;
				
			  load_address<=0;
			  load_command<=0;
			  load_data<=0;
			  write_en<=0;
			  read_en<=0;
			  tx_req<=0;
			  rx_finish<=0;
		end
		else begin
			load_address<=0;
			load_command<=0;
			load_data<=0;
			write_en<=0;
			read_en<=0;
			tx_req<=0;
			rx_finish<=0;
		
		
			case(state)
				IDLE:begin
					if(ssn_negedge)begin
						receive_data_counter_reset<=1;
						state<=WAIT_HEADER;
					end
					
				end
				
				WAIT_HEADER:begin
					if(sclk_posedge&&receive_data_counter==15)begin
						load_address<=1;
						load_command<=1;
					end
					else if(receive_data_counter==16)begin
						if(command)begin//read mode
							read_en<=1;
							tx_req<=1;
							rx_finish<=1;
							state<=IDLE;
							
						end
						else begin//write mode
							write_en<=1;
							state<=WAIT_DATA
						end
						
					end
					
				end	
				
				WAIT_DATA:begin
					if (sclk_posedge && receive_data_counter == 15) begin
									load_data <= 1'b1;
					end
					else if (receive_data_counter == 16) begin
									write_en  <= 1'b1;
									rx_finish <= 1'b1;
									state     <= IDLE;
					end
				end
				
				
			endcase
		end
	 end
endmodule

```

shift register

主要負責的功能: 
第一步先載入要傳送的data，接著當sclk_negedge時每次右移一位，並傳送MSB，轉成串列訊號從slave送到master。





```
module spi_tx_shift_register (
    input  wire        CLK_100MHz,
    input  wire        reset,
    input  wire [15:0] data,
    input  wire        load_shift_data,
    input  wire        sclk_negedge,
    output wire        miso,
    output reg  [15:0] shift_data
);
    always @(posedge CLK_100MHz) begin
        if (reset) begin
            shift_data <= 16'd0;
        end else if (load_shift_data) begin
            // 載入準備發送的資料
            shift_data <= data;
        end else if (sclk_negedge) begin
            // 配合 SPI Mode 0，在負緣移位
            shift_data <= {shift_data[14:0], 1'b0};
        end
    end

    // MISO 永遠輸出 shift_data 的最高位元
    assign miso = shift_data[15];

endmodule
```

TX_fsm

狀態轉換圖:
![image](https://github.com/Ericfishcan666/Reconfigurable-Chip/blob/main/9.png)

主要負責的功能:
主要分為三大區塊:
1.狀態轉移:
在每次100MHz_CLK來的時候，進行更新狀態轉換或重制狀態。

2.狀態轉移邏輯:
單純看現在的狀態來決定下一個狀態應該是甚麼，但是不負責更新。
INIT:
處於剛被reset的狀態。

START_SPI_TX:
待機狀態，直到收到來自RX的tx_req訊號才開始傳輸資料，否則一直在自己的狀態等待。

SEND_DATA:
開始傳輸，直到已經傳輸完16個bits。

FINISH:
回到初始狀態。

3.狀態控制邏輯:
根據現在的狀態來決定需要設定那些訊號，讓其他元件進行reset或load。
START_SPI_TX:
reset data counter並且把新的資料load進來

SEND_DATA:
剛開始進來reset data counter=0，所以shift register會持續運作，開始左移剛剛load進來的資料，並且在每個CLK送出MSB(miso)。
直到傳輸完畢才又reset data counter，為下一次的傳輸做準備。

```
module spi_tx_fsm (
    input  wire       CLK_100MHz,
    input  wire       reset,
    input  wire       tx_req,
    input  wire [4:0] send_data_counter,
    
    output reg        send_data_counter_reset,
    output reg        load_shift_data
);

    // 狀態定義 (Standard Verilog Parameter)
    localparam INIT         = 2'd0;
    localparam START_SPI_TX = 2'd1;
    localparam SEND_DATA    = 2'd2;
    localparam FINISH       = 2'd3;

    reg [1:0] current_state, next_state;

    // 1. 狀態更新 (Sequential)
    always @(posedge CLK_100MHz) begin
        if (reset) begin
            current_state <= INIT;
        end else begin
            current_state <= next_state;
        end
    end

    // 2. 下一個狀態邏輯 (Combinational)
    always @(*) begin
        case (current_state)
            INIT: 
                next_state = START_SPI_TX;
                
            START_SPI_TX: 
                if (tx_req == 1'b1) 
                    next_state = SEND_DATA;
                else 
                    next_state = START_SPI_TX;
                    
            SEND_DATA: 
                if (send_data_counter >= 5'd16) 
                    next_state = FINISH;
                else 
                    next_state = SEND_DATA;
                    
            FINISH: 
                next_state = INIT;
                
            default: 
                next_state = INIT;
        endcase
    end

    // 3. 輸出控制邏輯 (Combinational，嚴格對應狀態圖內部文字)
    always @(*) begin
        // 預設輸出為 0，避免產生 Latch
        send_data_counter_reset = 1'b0;
        load_shift_data         = 1'b0;

        case (current_state)
            START_SPI_TX: begin
                // 狀態圖: if (tx_req) send_data_counter_reset = 1, load_shift_data = 1
                if (tx_req == 1'b1) begin
                    send_data_counter_reset = 1'b1;
                    load_shift_data         = 1'b1;
                end
            end

            SEND_DATA: begin
                // 狀態圖: if (send_data_counter >= 16) send_data_counter_reset = 1
                if (send_data_counter >= 5'd16) begin
                    send_data_counter_reset = 1'b1;
                end
            end
            
            // INIT 與 FINISH 狀態無特別輸出動作
        endcase
    end

endmodule
```
